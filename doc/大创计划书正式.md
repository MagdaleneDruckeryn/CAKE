# 大创计划书：基于参数化特征的钢琴单音发生器

## （一）项目创意来历及项目意义（建议500字）

### 项目构思来历

本项目源于传统钢琴基础教学中长期存在的“模糊性”困境。在钢琴单音教学场景中，“音准”“音色”“触键”等核心概念的教学长期依赖教师的主观经验，教师常用“听起来偏亮”“触键太硬”“音准差一点”等模糊术语进行指导，学生难以精准理解问题本质，也缺乏可量化的练习目标。现有钢琴教学辅助工具（如电子调音器、普通发声软件、AI陪练软件）大多只能识别“对错”，无法拆解单音的物理特征，更无法定义“标准单音”的参数标准。

针对这一痛点，本项目提出“钢琴单音特征参数化+精准合成”的思路，旨在从真实钢琴单音中提取可量化的核心特征（如基频偏差、泛音能量占比、ADSR包络参数、触键力度对应分贝值等），建立“正确单音”的参数标准库，并基于参数合成高匹配度的单音，从而将钢琴单音教学从“经验驱动”转向“数据驱动”。

### 目的、意义等

**项目目的**：
1.  破解音乐教学的“模糊语言困境”，将抽象的声音评价术语转化为可定义、可测量的参数；
2.  建立钢琴单音的标准化参数体系，实现单音特征可量化、可区分、可复现；
3.  实现高还原度的单音合成与精准教学，让学生能够直接听到“标准音”，并通过参数差异直观看到问题所在；
4.  降低音乐入门门槛，提升教学效率，减少因语言理解偏差造成的反复试错。

**项目意义**：
-   **对音乐教学**：推动从“经验教学”走向“精准教学”，让教学评价更客观、统一、可复制；
-   **对学生**：帮助学生从“听不懂”到“看得见、听得懂、改得了”，实现自主、高效、有方向的练习；
-   **对技术与工具**：填补现有AI音乐教学的底层短板，为音准训练、音色训练、发声与触键教学提供底层技术支撑；
-   **对普及音乐教育**：让专业训练更普惠、更低门槛，推动音乐基础训练标准化、大众化。

### 国内外研究现状和发展动态

目前市场上已出现多类AI音乐教学产品，但均存在明显局限：

1.  **rhythm trainer**：仅聚焦节奏训练，无音高、旋律、和声、乐理等核心音乐能力训练，识别依赖屏幕点击，无法识别真实乐器演奏。
2.  **唱鸭app专业版**：主打“自弹自唱”与趣味创作，发声方法、气息控制等专业声乐教学薄弱，AI监测侧重音准、节奏，对音色、气息稳定性分析有限。
3.  **小叶子智能陪练**：AI对错音、节奏、力度、踏板的识别误判率高，教学逻辑机械，缺乏乐理讲解、指法逻辑、音乐处理的引导。
4.  **Earmaster/auralia**：侧重古典/传统视唱练耳体系，界面复杂、上手门槛高，教学互动性弱，缺乏游戏化、沉浸式设计。
5.  **TonalEnergy**：核心功能单一，主打音高可视化、调音、节拍器，无乐理教学、曲目陪练、视唱练耳等完整教学能力。

**共性缺陷**：
- 识别技术瓶颈：对复杂演奏、弱力度、多声部、环境噪音的识别精度不足；
- 教学深度不足：多为技能训练/纠错工具，缺乏乐理、音乐理解、情感表达等核心教学内容；
- 硬件与环境依赖：普遍依赖麦克风质量、外接设备、安静环境；
- 商业化问题：付费门槛高、核心功能锁定、售后体验差；
- 缺乏人文与个性化：AI反馈标准化、机械化，无法因材施教。

### 需求分析

当前AI音乐教学产品虽已覆盖节奏、声乐、乐器陪练、视唱练耳、音准可视化等场景，但均停留在“识别、纠错、评分、记录”层面，严重缺乏对声音最底层的“参数化解构、标准定义与可控合成”能力，形成显著的市场与技术空白：

1.  **现有产品只“判断对错”，不定义“什么是对”**：无法用可复现、可调节的参数定义一个标准、正确的单音。
2.  **只做“检测反馈”，不做“声音源头供给”**：不具备可控单音生成能力，无法根据教学需求精准输出对比样本。
3.  **参数化只用于“显示”，不用于“教学与合成”**：学生看到图表依然不懂如何调整，无法实现“参数—听觉—动作”的闭环教学。
4.  **底层能力缺失，无法支撑真正的精准音乐教育**：没有可参数化控制的标准单音发生器，所有音准训练、音色教学都只能停留在经验描述与模糊反馈。

因此，面向钢琴/声乐基础教学的“参数化标准单音发生器”是当前市场的明确空白，具有极强的研发价值与市场存在必要性。

## （二）项目研究主要内容（建议1000-2000字）

### 项目主要内容简介，要解决的问题（300字左右）

本项目旨在开发一套“可控钢琴音色生成系统”，通过参数化方式解构钢琴单音的特征，并能够根据预设参数合成与真实钢琴高度匹配的单音。核心解决以下问题：

1.  **传统音乐教学术语模糊问题**：将教师口中“音太飘”“音色太暗”“音不准”等抽象描述，拆解为音高、响度、起音速度、泛音结构、音高波动范围等可定义参数，建立统一、可执行的量化标准。
2.  **单音特征量化与标准缺失问题**：从真实钢琴音色中提取关键声学特征，明确“正确单音”应满足的参数范围，实现单音特征可量化、可区分、可复现。
3.  **合成音色与真实音色脱节问题**：基于提取的参数合成与真实钢琴高度匹配的单音，为学生提供稳定、统一的听觉参照，辅助精准教学。
4.  **现有工具功能局限问题**：弥补现有AI音乐教学软件“仅识别对错、不量化特征”的短板，为钢琴基础教学工具提供参数化的技术范式。

### 概述项目研究方法，要解决的问题（比如：解决方案，架构设计）

**解决方案**：
项目采用“特征提取→参数标准化→音色合成→前端交互”的技术路线。
1.  **特征提取**：采集真实钢琴单音样本，通过信号处理技术提取基频、泛音结构、ADSR包络、力度-响度映射等核心特征参数。
2.  **参数标准化**：对不同音高、不同触键方式下的特征参数进行统计分析，建立“标准单音”的参数范围库。
3.  **音色合成**：利用物理建模或样本拼接合成技术，开发能够根据输入参数生成对应单音的合成引擎。
4.  **前端交互**：开发Web前端界面，提供参数调控、音色生成、音频试听、A/B对比、可视化图表等功能，形成完整的教学工具。

**架构设计**：
系统采用前后端分离架构。前端负责参数交互、音频播放、可视化渲染；后端负责音色合成算法、参数处理、音频流生成。前后端通过HTTP/WebSocket协议通信，实现参数提交、音频流接收、状态同步等功能。

## （三）项目创新点与项目特色（建议200字左右）

1.  **参数化单音解构**：首次将钢琴单音教学中的模糊术语转化为可量化、可调节的声学参数，建立“标准单音”的参数体系，实现教学评价的客观化、标准化。
2.  **可控单音合成**：开发能够根据预设参数生成高匹配度钢琴单音的合成引擎，提供“声音源头供给”，弥补现有工具只检测不生成的短板。
3.  **教学闭环设计**：集成参数调控、标准音生成、A/B对比、可视化图表等功能，形成“参数设置→声音生成→听觉对比→视觉反馈”的完整教学闭环，帮助学生直观理解参数与音色的关联。
4.  **底层技术支撑**：项目聚焦于音乐教学的底层技术设施，为AI音乐教育提供可参数化控制的标准单音发生器，支撑更精准的音准、音色、触键教学。

## （四）系统方案和技术路线（建议1000字左右）

### 技术关键和设计思路

**技术关键**：
1.  实现**HTTP/WebSocket双协议**的前后端接口通信，保障音色参数θ的稳定提交和wav/音频流的高效接收，同时实时同步后端生成状态；
2.  打造高精度的参数交互控件，支持音高（C4–B5）、力度（0–127/0–1）及包络、音色相关参数的精准调控，且实现参数实时数值反馈；
3.  完成前端音频处理能力，支持生成音频的播放、暂停、重播及时长展示，实现A/B音色快速切换对比的音频逻辑；
4.  实现可视化图表渲染，支持包络曲线、谐波分布柱状图的动态展示，匹配参数调整的实时视觉反馈。

**设计思路**：
1.  **交互轻量化**：以钢琴音色生成的专业参数为核心，设计贴合音乐从业者操作习惯的控件（虚拟钢琴键盘、Slider等），搭配tooltip参数说明，降低操作门槛；
2.  **模式化设计**：支持“音色生成/教学对比”双模式切换，兼顾专业音色生成需求和教学场景的可视化、对比需求；
3.  **状态可视化**：对后端处理、音频加载、参数调整等全流程状态进行显性化展示（Loading、处理中/完成、数值实时变化），提升操作反馈感；
4.  **多场景适配**：兼顾日常使用和演示需求，设计Demo固定参数模式及截图/录屏友好布局，保障项目展示效果；
5.  **前后端解耦**：前端专注于参数交互、音频展示、可视化渲染，通过标准化接口与后端/AI模型通信，提升系统扩展性。

2.  教学辅助可视化模块为**可选模块**，可根据项目落地需求灵活取舍，不影响核心音色生成功能的运行；
3.  系统演示支持模块为辅助模块，基于核心功能做定制化布局和参数固化，服务于项目展示、演示场景；
4.  所有模块均围绕“参数输入-后端交互-结果展示”的核心流程设计，各模块间通过状态共享、事件触发实现联动。

### 功能概述（含子模块）

**一、系统基础框架模块**：
1.  **系统运行入口子模块**：提供Web/桌面统一入口页面，展示项目名称和所有功能模块入口；支持“音色生成/教学对比”模式一键切换。
2.  **前后端接口通信子模块**：封装HTTP/WebSocket接口，实现前端音色参数θ向后端的提交；接收后端返回的wav音频文件或音频流；实时获取并展示后端音色生成状态。

**二、音色参数交互模块**：
1.  **音高控制子模块**：提供虚拟钢琴键盘/下拉菜单两种选择方式，限定调节范围为C4–B5；选择后实时显示对应音名及频率。
2.  **力度控制子模块**：通过Slider控件实现力度调节，支持0–127或0–1两种数值范围；滑块拖动过程中实时显示当前力度数值。
3.  **核心音色参数控制子模块**：提供起音时间、衰减速度、明亮度、谐波强度（简化维度）的可调控件；每个参数搭配tooltip悬浮提示。

**三、音色生成与试听模块**：
1.  **单音生成触发子模块**：提供单音生成按钮，点击后触发后端AI模型的音色生成逻辑；按钮及界面同步显示Loading状态。
2.  **音频播放控制子模块**：对接后端返回的音频资源，提供播放、暂停、重播核心操作；解析音频文件并展示音频时长。
3.  **A/B音色对比子模块**：支持标准钢琴音色与当前生成音色的双轨加载；提供快速切换按钮，实现两种音色的无缝试听对比。

**四、教学辅助可视化模块（计划）**：
1.  **包络曲线显示子模块**：根据起音时间、衰减速度等参数，实时绘制并展示音色的包络曲线。
2.  **谐波分布柱状图子模块**：基于谐波强度等参数，生成谐波分布柱状图，展示不同谐波的强度占比。
3.  **参数差异文字提示子模块**：对比当前生成音色与标准音色的参数数值，以文字形式展示参数差异。

**五、系统演示支持模块**：
1.  **Demo演示模式子模块**：提供固定参数的演示模式，启动后无需手动调节参数，一键触发预设音色的生成和试听。
2.  **展示布局优化子模块**：对系统界面进行截图、录屏友好化设计，优化控件布局、状态展示的视觉效果。

```
        └─ 视图层（前端页面）

            ├─ 可视化视图：包络曲线、谐波柱状图（可选）
            └─ 演示视图：Demo模式入口、优化布局展示
                └─ 逻辑层（前端核心逻辑）
                    ├─ 状态管理：全局状态（生成状态、模式状态）、参数状态、音频状态
                    ├─ 接口通信：HTTP/WebSocket封装、参数提交、音频接收、状态同步
                    ├─ 音频处理：音频资源解析、播放控制、A/B双轨管理
                    ├─ 可视化渲染：图表初始化、参数联动刷新、数据可视化转换
                    └─ 演示逻辑：固定参数固化、布局适配逻辑
                        └─ 接口层
                            ├─ 参数提交接口（HTTP/WebSocket）
                            ├─ 音频资源接收接口（HTTP/WebSocket）
                            └─ 生成状态查询接口（HTTP/WebSocket）
                                └─ 后端/AI模型层
                                    ├─ 接收前端参数θ
                                    ├─ 执行钢琴音色生成算法
                                    ├─ 返回wav/音频流
                                    └─ 同步生成处理状态
```

**架构说明**：
1.  整体采用**分层架构设计**，从用户层到后端/AI模型层逐层向下交互，每层职责单一，便于开发、维护和扩展；
2.  **视图层与逻辑层解耦**：视图层仅负责界面渲染和用户操作捕获，业务逻辑、数据处理全部交由逻辑层实现；
3.  **接口层做标准化封装**：对前后端的所有交互接口进行统一封装，屏蔽协议差异（HTTP/WebSocket）；
4.  **状态管理为核心**：逻辑层通过全局状态和局部状态管理，实现各视图、各模块间的状态联动；
5.  架构支持**可选模块灵活集成**：教学辅助可视化的图表渲染逻辑独立于核心逻辑，可按需集成。

### 技术亮点（比如：算法改进、算法创新等）

1.  **参数化音色合成算法**：采用物理建模或高质量样本拼接技术，实现根据多维参数实时生成高保真钢琴单音，保证合成音色与真实钢琴的高度匹配。
2.  **双协议通信优化**：结合HTTP与WebSocket，兼顾参数提交的可靠性与音频流、状态同步的实时性，提升系统交互体验。
3.  **前端音频双轨管理**：实现标准音色与生成音色的快速切换与对比，解决音频播放中的卡顿、延迟问题，保障教学对比的流畅性。
4.  **参数-图表实时联动**：基于前端可视化库，实现参数调整与包络曲线、谐波分布图的实时刷新，直观展示参数对音色的影响。

### 开发难度

**低难度模块**：
- 系统运行入口模块：页面布局、导航设计、模式切换，基于UI组件库可快速实现。
- 系统演示支持模块：Demo模式参数固化、布局优化，无复杂业务逻辑。

**中等难度模块**：
- 力度/音高参数交互模块：需定制虚拟钢琴键盘、Slider控件，实现参数范围限定与实时反馈。
- 单音生成与基础音频播放模块：触发接口请求、展示Loading状态，基于Web Audio API实现播放控制。
- 教学辅助可视化模块（可选）：基于可视化库开发图表，实现参数与图表的联动刷新。

**较高难度模块**：
- 前后端接口通信模块：需同时支持HTTP/WebSocket协议，处理参数提交、音频流接收、状态同步，并考虑网络异常、请求中断等容错处理。
- 核心音色参数控制模块：需对接专业钢琴音色参数，实现控件与参数的精准映射，设计友好tooltip，兼顾专业性与易用性。
- A/B音色对比模块：需实现双轨音频管理，支持快速无缝切换，处理音频缓存、播放状态同步、音色切换无卡顿。

**整体开发难度**：中等。核心难点集中在接口通信、音频处理、参数交互控件定制三个较高难度点；可选模块和辅助模块可分摊开发压力，采用“先核心后辅助、先基础后专业”的开发节奏，可保障开发进度。

### 达到的技术指标

1.  **参数调节范围**：
    - 音高：C4–B5（约261.63 Hz–987.77 Hz）；
    - 力度：0–127（MIDI标准）或0–1（归一化）；
    - 起音时间：0–200 ms可调；
    - 衰减速度：0–500 ms可调；
    - 明亮度：0–100%可调；
    - 谐波强度：0–100%可调。
2.  **音色合成性能**：
    - 生成单音与真实钢琴单音的感知相似度（主观评测）达到85%以上；
    - 单音生成延迟（从参数提交到音频可播放）小于500 ms；
    - 支持实时参数调整与音色实时刷新。
3.  **系统兼容性**：
    - 支持Chrome、Firefox、Edge等主流浏览器；
    - 界面适配桌面端与平板设备。
4.  **教学辅助功能**：
    - 实现包络曲线、谐波分布图的实时绘制；
    - 支持标准音色与生成音色的无缝A/B对比。
